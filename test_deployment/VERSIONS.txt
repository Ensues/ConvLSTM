================================================================================
                         MODEL SPECIFICATIONS
================================================================================

Model Architecture: ConvLSTM Prototype 10
  - Input Shape: [1, 20, 6, 128, 128]
    * Batch size: 1
    * Sequence length: 20 frames
    * Channels: 6 (3 RGB + 3 Intent)
    * Frame size: 128x128 pixels
  
  - Output: [1, 3] probabilities for Front/Left/Right
  
  - Size: ~1.5 MB (TFLite format)
  
  - Features:
    * 2-layer ConvLSTM (hidden_dim: [64, 32])
    * Global Average Pooling (80% model size reduction)
    * Dropout (0.5) for regularization

================================================================================
                           BUILD MODES
================================================================================

1. EXPO GO (Demo Mode)
   Command: npx expo start
   - Quick testing on device using Expo Go app
   - TFLite inference NOT available (native module)
   - Runs with SIMULATED predictions
   - All UI and camera features work normally

2. DEVELOPMENT BUILD (Full Features)
   Commands:
     npx expo prebuild
     npx expo run:android
   - Builds native Android app with all modules
   - TFLite inference ENABLED
   - Real model predictions
   - Recommended for testing actual model performance

3. PREVIEW BUILD (APK Distribution)
   Command: npx eas build --platform android --profile preview
   - Creates installable APK
   - Full TFLite support
   - For internal testing and distribution

4. PRODUCTION BUILD
   Command: npx eas build --platform android --profile production
   - Optimized AAB for Play Store
   - Code signing and obfuscation
   - Full TFLite support

================================================================================
                     WHY THESE SPECIFIC VERSIONS?
================================================================================

STABILITY VS CUTTING-EDGE BALANCE:
- Expo SDK 54 is the latest stable, ensuring all plugins are compatible
- React 19 provides performance improvements for real-time processing
- TFLite 1.6.1 is thoroughly tested with Expo and supports GPU delegates

EXPO MANAGED WORKFLOW:
- Easier dependency management than bare React Native
- Automatic native code generation via expo prebuild
- EAS Build for cloud-based CI/CD

MODEL INFERENCE:
- react-native-fast-tflite chosen over alternatives (tflite-react-native, 
  onnx-runtime-react-native) for best Expo compatibility and performance

================================================================================
                              KNOWN LIMITATIONS
================================================================================

1. Frame Capture Speed:
   - takePictureAsync is ~200-500ms per frame (~2-3 FPS)
   - For 20 FPS, would need react-native-vision-camera + frame processors
   - Current implementation duplicates frames to fill buffer

2. Base64 Decoding:
   - No direct pixel access in Expo Go
   - Using statistical sampling from JPEG compressed data
   - Dev build with expo-gl could provide true pixel extraction

3. iOS Support:
   - Not actively tested (Android primary target)
   - Should work with minor adjustments
